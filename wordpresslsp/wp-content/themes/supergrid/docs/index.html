HTML Structure

This grid-based theme stores all your content in "Widgets". the starter.html file contains the basic markup for the grid without the widgets. You can quickly and easily add content by placing widgets in the "#container" div.

Here is the markup for a basic widget:

<div class="widget portfolio">
  <div class="entry-container span4">
    <div class="entry drop-shadow curved six-col">

      // CONTENT GOES HERE

      <div class="stripes"></div>
    </div>
  </div>
</div>

This theme comes with many pre-built widgets. The easy to use widgets also allow you to create your own custom widgets to display anything you like. To see how to implement these widgets into the theme, please see the Widget section.

Alternatively you can use the index.html which is already populated with Widgets and modify and customise it to your preference.

Navigation - Drop Down

Here's a look at the basic html structure of the navigation with a drop down menu.

<div id="nav-collapse" class="collapse out">
  <ul class="nav">

    <li>
      <a href="#filter=.portfolio" class="selected">
        <span data-toggle="collapse" data-target="#blog">
        </span>BLOG
      </a>
      <ul id="blog" class="collapse out">
        <li><a href="#filter=.graphics">GRAPHICS</a></li>
        <li><a href="#filter=.illustration">ILLUSTRATION</a></li>
        <li><a href="#filter=.web">WEBDESIGN</a></li>    
      </ul>
    </li>
    <li><a href="#filter=.grid-widget">GRID WIDGETS</a></li>
    <li><a href="#filter=.shortcodes">ELEMENTS</a></li>
    <li><<a href="#filter=.contact">CONTACT</a></li>

  </ul>
</div>

What you can see here is basically another UL placed within an LI to create a drop down. A span element must also be added in the main a element, for the the responsive state. This span basically displays the drop down "on click" rather than "on hover"

Heads up! Don't forget to apply a data-target in the span and reference it in the UL id.

Navigation - Filter

This theme uses a filter based navigation.

How does it work?

Every widget must have a category. This category is used by the navigation to filter the widgets.

This widget has the category "portfolio"

<div class="widget portfolio">
  <div class="entry-container span8">
    <div class="entry drop-shadow curved six-col">

      // CONTENT GOES HERE

      <div class="stripes"></div>
    </div>
  </div>
</div>

When this portfolio navigation button is clicked, it will filter all the widgets and display only the ones with the category "portfolio".

<li><a href="#filter=.portfolio">PORTFOLIO</a></li>

Multiple Categories

You can place as many categories as you want in the navigation or widget

<li><a href="#filter=.portfolio,.about">PORTFOLIO</a></li>
_______

<div class="element portfolio about contact">

Homepage

All widgets you want displayed on the homepage must have that "homepage" category

Widgets

All content on this theme is handled by Widgets or "Gridgets" (Grid Widgets). This theme comes with multiple ready built widgets.

The markup for all the widgets is available in docs/widgets.html
You can either populate your own grid by using the Widget mark-up in 'docs/widget.html' and placing it in the 'starter.html' or you can use the already populated grid located in 'index.html'

Custom Widgets

This theme is very versatile. The possibilities for widgets are endless. Using the basic widget mark up:

<div class="element portfolio">
  <div class="entry-container span4">
    <div class="entry drop-shadow curved six-col">

      // CONTENT GOES HERE

      <div class="stripes"></div>
    </div>
  </div>
</div>

To adjust the size of the widget, change the span number. A double sized widget would have a span8.

You can create many different types of widgets:

  • Facebook widgets
  • Twitter widgets
  • Instagram widgets

All you have to do is use the basic widget mark up and add in your own content or embed codes

Typography

For a simple two column layout, create a .row-fluid and add the appropriate number of .span* columns. As this is a 12-column grid, each .span* spans a number of those 12 columns, and should always add up to 12 for each row (or the number of columns in the parent).

<div class="row-fluid">
  <div class="row">
    <div class="span4">...</div>
    <div class="span8">...</div>
  </div>
</div>

Google Maps API

The 'Contact' page widget is powered by Google's powerful API.

To active your maps you must sign up for Google Maps API and insert your API key into the header script file

<script type="text/javascript" src="https://maps.googleapis.com/maps/api/js?key=YOUR_KEY_GOES_HERE&sensor=true"></script>

All settings for the API are located in 'js/maps.js'
